how credits work, in words

A new mind gets a wallet at creation. Its Cognitive Credits = wallet balance of supported tokens: 1 MENTE = 1 credit; MOCA counts by USD value (USDC support planned). Holding MOCA with no MENTE triggers a small auto-swap to MENTE (the actionable credit). Each user request makes the mind pay MENTE to the collector — gasless for the user: the gas funder first sends a sliver of ETH to the mind wallet in the same moment (visible on-chain as paired transactions). Value leaves the loop via an automatic contract burn (~1.2% lifetime, being confirmed) and enters via Stripe purchases and manual treasury funding.

AI-inferred interpretation

Everything in this amber zone is inferred — from transfer sizes, timing heuristics, or projections — not read from a ledger. Each block states its method. Treat as triage, not truth.
AI-inferredPayout type classification — invokes, equips, credits, Stripe

Method: each transfer is snapped to the platform's known price points ($0.10 invoke, $1 equip, $3 credit, $5 referral, $10/25/50 Stripe) by its USD value at that day's token price, ±8% — the same tolerance as the factual size bands above, so band counts and type counts always agree. This is inference from size, not ground truth: a per-transfer memo from the payout contract (requested) would make it exact. The $3 bucket mixes new-user credits and internal top-ups — the internal top-ups sheet is the authoritative split. Stripe-sized transfers are deliveries of fiat purchases (revenue-backed); confirming them requires the Stripe ledger, which this page does not read yet.

AI-inferredCreator earnings ranking (classified invokes + equips)

Method: recipients of transfers classified as invoke/equip, ranked by USD, share of ALL classified creator earnings. Inherits the size classification's uncertainty.

AI-inferredPayout pattern monitor

Method: we look at each earner's payout timing and ask: does it look human-irregular or robotically regular? Three signals: timing variety, repeating rhythm, and rapid-fire bursts (technical: gap entropy, lag-1 autocorrelation, 60-second burst share). Cut-offs are judgment calls; flags mean "worth a look", never "confirmed abuse". Automated invocation can be legitimate — the automation policy is an open decision. Flagged rows sort first.
